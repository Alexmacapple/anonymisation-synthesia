Principe de fonctionnement

L'outil détecte automatiquement les données personnelles (PII) dans vos fichiers et les remplace par des jetons anonymes ([PERSONNE_1], [EMAIL_1], etc.).

Il combine deux moteurs de détection complémentaires :

Les deux moteurs de détection
MoteurComment ça marcheForceLimite
Regex + dictionnaires Patterns fixes (email, téléphone, IBAN) + 884 000 noms de famille + 169 000 prénoms Rapide, déterministe, validateurs mathématiques (Luhn, mod-97) Ne comprend pas le contexte ("Rose" = prénom ou fleur ?)
GLiNER2 (NER) Modèle d'intelligence artificielle qui comprend le contexte sémantique Détecte les noms dans le texte libre ("Alexandra m'a proposé…") Plus lent, peut avoir des faux positifs

Le mode hybrid (par défaut) utilise les deux moteurs simultanément et fusionne les résultats pour une couverture maximale.

Exemple concret

Texte original : "Bonjour, je suis Pierre Durand, email pierre.durand@example.com. Alexandra m'a proposé un tarif."

Résultat : "Bonjour, je suis [PERSONNE_1], email [EMAIL_1]. [PERSONNE_2] m'a proposé un tarif."

"Pierre Durand" et "pierre.durand@example.com" sont détectés par les deux moteurs. "Alexandra" n'est détectée que par GLiNER2 (NER) — les regex l'auraient ratée.

Installation

# Cloner le dépôt
git clone [url-du-depot]
cd anonymisation-synthesia

# Installer les dépendances (crée le venv)
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# Lancer le serveur
.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8091

Au premier lancement, le modèle GLiNER2 (~800 Mo) est téléchargé automatiquement depuis HuggingFace. Les lancements suivants utilisent le cache local.

Premiers pas (30 secondes)

  1. Ouvrez l'onglet Pseudonymisation
  2. Collez un texte contenant des données personnelles dans le champ de gauche
  3. Cliquez sur Pseudonymiser
  4. Le texte anonymisé apparaît à droite avec un score RGPD

Pour traiter un fichier complet (JSON, CSV, XLSX, DOCX, PDF) :

  1. Ouvrez l'onglet Import fichier
  2. Choisissez votre fichier (upload ou chemin local)
  3. Pour les fichiers structurés (JSON, CSV) : cliquez sur Générer le mapping pour configurer les champs à anonymiser
  4. Cliquez sur Traiter le fichier
  5. Téléchargez le résultat anonymisé et les correspondances CSV

Modes de détection

En une phrase

  • Regex : cherche des patterns fixes (un email c'est xxx@xxx.xx, un téléphone c'est 06 xx xx xx xx). Rapide, fiable sur les formats connus, mais ne comprend rien au texte.
  • NER : un cerveau artificiel (GLiNER2) qui lit le texte et comprend que "Alexandra m'a proposé un tarif" → Alexandra est une personne. Il comprend le contexte, mais il peut se tromper.
  • Hybrid : les deux ensemble. Le regex attrape les emails/tel/IBAN avec certitude, le NER attrape les noms cachés dans les phrases.

Ligne de commande (CLI)

# Pseudonymiser un texte brut (DOCX, PDF, TXT — pas besoin de mapping)
.venv/bin/python3 cli.py document.docx --pseudo --mode hybrid

# Pseudonymiser un fichier structuré avec mapping
.venv/bin/python3 cli.py export.json --mapping mapping.json --pseudo --mode hybrid

# Générer un mapping automatiquement
.venv/bin/python3 cli.py export.json --mapping-generate

# Aperçu sur 100 enregistrements (dry-run)
.venv/bin/python3 cli.py export.json --mapping mapping.json --dry-run

# Scoring RGPD sans anonymiser
.venv/bin/python3 cli.py export.json --mapping mapping.json --score-only

# Mode fort + détection technique
.venv/bin/python3 cli.py fichier.json --mapping mapping.json --pseudo --fort --tech

# Traiter un dossier entier
.venv/bin/python3 cli.py --input-dir dossier/ --mapping mapping.json --pseudo

# Choisir le mode de détection
--mode regex    # Regex seul (rapide)
--mode ner      # GLiNER2 seul (contextuel)
--mode hybrid   # Les deux combinés (défaut)

Glossaire

TermeDéfinition
PIIPersonally Identifiable Information — toute donnée permettant d'identifier une personne (nom, email, téléphone, IBAN, etc.)
PseudonymisationRemplacement des données par des jetons ([PERSONNE_1]) — réversible avec les correspondances
AnonymisationRemplacement irréversible par *** — les données originales sont perdues
NERNamed Entity Recognition — détection d'entités nommées par intelligence artificielle
GLiNER2Modèle NER zero-shot (205M paramètres) utilisé pour la détection contextuelle
MappingFichier JSON décrivant la structure du fichier à traiter (champs sensibles, texte libre, whitelist)
Score RGPDNote de risque calculée selon les types de données détectées (direct, finance, technique, indirect)
CorrespondancesTable associant chaque jeton à sa valeur originale (permet la restauration)
HybridMode combinant regex et NER pour une couverture maximale

Questions fréquentes

Non. Tout le traitement s'exécute sur votre machine. Le modèle GLiNER2 est téléchargé une seule fois au premier lancement, puis fonctionne hors ligne. Aucune donnée personnelle ne transite par internet.

La pseudonymisation remplace les données par des jetons ([PERSONNE_1]) et sauvegarde les correspondances dans un fichier CSV. Vous pouvez restaurer les données originales à tout moment.

L'anonymisation remplace les données par ***. C'est irréversible — les données originales sont perdues.

Le mapping indique à l'outil quels champs contiennent des données personnelles et comment les traiter. Sans mapping, l'outil ne sait pas que le champ Report.Firstname contient un prénom.

Pour les documents non structurés (DOCX, PDF, TXT), le mapping n'est pas nécessaire — tout le texte est scanné automatiquement.

Cliquez sur Générer le mapping automatiquement pour obtenir une proposition basée sur la structure du fichier.

Le mode hybrid (défaut) est recommandé pour la plupart des cas. Il combine les forces des deux moteurs.

Utilisez regex seul si vous avez un très gros fichier (> 10 000 enregistrements) et que la vitesse est prioritaire.

Utilisez NER seul pour du texte libre avec des noms ambigus que les regex pourraient rater.

Utilisez l'onglet Diagnostic > Vérifier un texte anonymisé. Collez le texte déjà anonymisé et l'outil repassera GLiNER2 dessus pour détecter les éventuelles fuites de données personnelles.

Oui. Au premier appel NER, le modèle GLiNER2 (~800 Mo) est chargé en mémoire. Cela prend 5 à 30 secondes selon la machine. Les appels suivants sont rapides (~0.2s par texte).

Gouvernement

Outil d'anonymisation PII 100 % local. Combine GLiNER2 (NER contextuel) et regex avec validateurs mathématiques. Aucune donnée personnelle ne quitte la machine. Le traitement s'exécute intégralement sur votre poste.
